## 02 · 🎨 Formatting Tips

> **Make your spreadsheets look clean, professional, and presentation-ready**

| # | 📌 Tip | 💡 How To Do It |
|---|---|---|
| 1 | **Hide gridlines** | Page Layout → uncheck View Gridlines |
| 2 | **Format Painter** | Copy the format of one cell to another quickly |
| 3 | **Clear Formats** | Remove all formatting to get clean raw data |
| 4 | **Remove formula errors** | Use `=IFERROR(formula, 0)` or `=IFERROR(formula,"")` |
| 5 | **Add leading zero** | Type `'` before the number e.g. `'0123` |
| 6 | **Freeze Panes** | Keep headers visible when scrolling down |
| 7 | **Consistent date format** | Always use the same date format throughout the file |
| 8 | **Lock cells with F4** | Press F4 once = lock row & column (`$F$9`), twice = lock row (`F$9`), three times = lock column (`$F9`) |
| 9 | **Expand column width** | Highlight columns → double-click between column headers |
| 10 | **Word Wrap** | Home → Word Wrap — stops text spilling into next cell |
| 11 | **Spell Check** | Review → Spelling |
| 12 | **Flash Fill** | `Ctrl + E` — auto-fills a pattern you start |
| 13 | **Input cell colour standard** | Blue background = input cell · Yellow = needs updating · White = formula cell |
| 14 | **Subtotal alignment trick** | Use cell alignment (not spaces) for indented subtotals — spaces break formulas |
| 15 | **Instructions tab** | Always add a sheet explaining how to use the model |
| 16 | **Assumptions tab** | Keep all key variables in one dedicated tab |
| 17 | **Date placement** | Put dates along columns, summarize into quarters and years |

**IFERROR Example:**
```excel
= IFERROR(D15/D14, 0)           ← Returns 0 if error
= IFERROR(D15/D14, "")          ← Returns blank if error
```

**Cell Lock Example (F4 Key):**
```excel
$F$9   ← Locks both column and row    (F4 once)
F$9    ← Locks row only               (F4 twice)
$F9    ← Locks column only            (F4 three times)
```

## 03 · 📅 Dates Formulas

> **Work with dates like a finance professional**

| # | 📌 Formula | 💡 Purpose |
|---|---|---|
| 1 | Drag corner to fill | Auto-populate sequential dates or numbers |
| 2 | `=B2 - A2` | Count days between two dates |
| 3 | `=TODAY()` | Insert today's date |
| 4 | `=COUNTIF(range,">"&cell)` | Count items more recent than a date |
| 5 | `=COUNTIFS(range,"<"&date, range,">"&date)` | Count items between two dates |
| 6 | `=SUMIFS(sum, range,">"&date)` | Sum items greater than a date |
| 7 | `=EOMONTH(date, 0)` | Get end-of-month date |
| 8 | `=EOMONTH(date, 1)` | Get end of next month |
| 9 | `=EOMONTH(date, -1)` | Get end of previous month |
| 10 | `="Q"&ROUNDUP(MONTH(date)/3,0)&" "&YEAR(date)` | Convert date to quarter text (e.g. "Q1 2020") |
| 11 | `=DAY(date)` | Return just the day number |
| 12 | `=MONTH(date)` | Return just the month number |
| 13 | `=YEAR(date)` | Return just the year number |
| 14 | `=TEXT(date,"dddd")` | Return day of week (e.g. "Monday") |
| 15 | `=TEXT(date,"mmmm")` | Return month name (e.g. "January") |

**Quarter Formula Example:**
```excel
="Q"&ROUNDUP(MONTH(A1)/3,0)&" "&YEAR(A1)
→ Result: "Q1 2020"
```

**EOMONTH Examples:**
```excel
=EOMONTH(A1, 0)   → Jan 31, 2020  (end of current month)
=EOMONTH(A1, 1)   → Feb 29, 2020  (end of next month)
=EOMONTH(A1,-1)   → Dec 31, 2019  (end of previous month)
```

## 04 · 📆 Workday Formula

> **Calculate real working days — excluding weekends and holidays**

**When to use WORKDAY:**
- Calculate delivery dates for shipments
- Calculate due dates for invoices
- Set project deadlines (skipping weekends)
- Any timeline that must exclude non-working days

**Formula Syntax:**
```excel
=WORKDAY(start_date, days, [holidays])
```

| Part | Meaning |
|---|---|
| `start_date` | The date to start from |
| `days` | Number of working days to add (positive = future, negative = past) |
| `holidays` | Optional list of holiday dates to also skip |

**Example — Delivery Date Calculator:**
```excel
=WORKDAY(order_date, shipping_days, holiday_range)

Order Date: Dec 15, 1998
Shipping Days: 28
Holiday (New Year Jan 1): excluded automatically
→ Result: Jan 26, 1999  ✅
```

## 05 · ⚠️ Common Errors

> **Understand and fix the 7 most common Excel error messages**

| ❌ Error | 🔍 What Went Wrong | ✅ How to Fix It |
|---|---|---|
| `######` | Column too narrow to show the value | Widen the column |
| `#DIV/0!` | Number divided by zero or blank cell | Use `IFERROR` or check for blanks first |
| `#NAME?` | Function name is misspelled | Fix the spelling of the formula name |
| `#VALUE!` | Formula mixes different data types | Make sure all inputs are the same type |
| `#REF!` | A cell referenced in formula was deleted | Update the formula to fix the broken link |
| `#NUM!` | Invalid number in formula (e.g. wrong date order in DATEDIF) | Review numeric inputs for validity |
| `#NULL` | Two ranges in formula don't overlap | Update the formula so ranges connect properly |

**Quick Fix — Wrap any formula with IFERROR:**
```excel
= IFERROR(your_formula, 0)      ← Replace error with 0
= IFERROR(your_formula, "N/A")  ← Replace error with text
= IFERROR(your_formula, "")     ← Replace error with blank
```

## 06 · ✂️ Copy vs. Cut

> **Know when to copy and when to cut — especially for formulas**

| Action | When to Use |
|---|---|
| **Copy** (`Ctrl + C`) | You want to duplicate content. Original stays in place. |
| **Cut** (`Ctrl + X`) | You want to **move** content. Original is deleted after paste. |

> ⚠️ **Important Rule for Formulas:**
> When **moving a formula** to a new cell, always use **CUT** — not copy.
> If you copy a formula and paste it to a new location, the formula still references the OLD location which causes incorrect results or `#REF!` errors.

**Example:**
```
✅ Correct: Cut =SUM(E31:E46) from D22 → Paste to D23
   Result: Formula now correctly sums in new location

❌ Wrong: Copy =SUM(E31:E46) from D22 → Paste to D23
   Result: Formula still exists in D22 AND D23 — double-counting!
```

## 07 · 🔍 Precedents & Dependents

> **Visually trace where formulas come from and where they go**

**What does it do?**
Excel draws blue arrows between cells to show you the relationships between formulas and their data sources.

| Term | Meaning |
|---|---|
| **Precedent** | The cell that a formula **reads from** (source) |
| **Dependent** | The cell that **uses** another cell's value (destination) |

**Steps to Use:**
1. Click on any cell you want to investigate
2. Go to **Formulas tab** → click **Trace Precedents** (shows where data comes from)
3. Go to **Formulas tab** → click **Trace Dependents** (shows where data goes to)
4. Blue arrows appear showing all connected cells
5. Click **Remove Arrows** when done

**When is this useful?**
- 🔎 Auditing a formula to understand where a number came from
- 🔒 Checking if a cell is used elsewhere before you delete it
- 🐛 Debugging broken formulas in complex workbooks
- 📊 Understanding multi-sheet formula relationships

## 08 · 📊 Group & Ungroup

> **Collapse and expand rows or columns for cleaner presentations**

**When to use:**
- You have a large spreadsheet and want a clean summary view
- Group monthly columns into quarters
- Group quarterly columns into yearly totals
- Group detailed rows under a subtotal row

**How to Group Rows:**
1. Select the rows you want to group *(do NOT include the subtotal row)*
2. Go to **Data tab** → click **Group**
3. A `+` / `−` button appears on the left side
4. Click `−` to collapse · Click `+` to expand

**How to Group Columns:**
1. Select the columns you want to group
2. Go to **Data tab** → click **Group**
3. A `+` / `−` button appears at the top

**Nested Groups (Outer + Inner):**
```
▼ TOTAL CASH                    ← Outer Group (collapse all)
  ▼ Cash On Hand                ← Inner Group
      Petty Cash - Alabama
      Petty Cash - California
      Petty Cash - Washington
  ▼ Cash In Banks               ← Inner Group
      Bank of America - CA
      JPMorgan - SA
```

**To Ungroup:** Select the rows/columns → Data tab → Ungroup

## 09 · 🔒 Protect Ranges

> **Lock cells, ranges and worksheets to prevent accidental changes**

**Why use it?**
When sharing a workbook with others, you can lock specific cells so only you can modify the formulas and key data — while still letting others fill in input cells.

**Steps to Protect a Sheet:**
1. Go to **Review tab** → click **Protect Sheet**
2. Choose what users can still do (select cells, sort, format, etc.)
3. Set a password → click **OK**

**Steps to Allow Editing in Specific Cells:**
1. Go to **Review tab** → click **Allow Edit Ranges** → click **New**
2. Give the range a name and highlight the cells users can edit
3. Optionally set a range password for those specific cells
4. Then protect the full sheet — the allowed cells remain editable

**To Protect Entire Workbook:** Review → Protect Workbook (prevents adding/deleting sheets)

> 💡 **Pro Tip:** You can allow some users to edit a specific range with their own password, while everyone else is locked out — great for shared finance models sent to multiple departments.

## 10 · 📸 Screenshot Function

> **Insert screenshots directly inside your Excel file**

**When to use:**
- Attaching bank balance screenshots for reconciliation
- Adding reference images to support a number in your model
- Creating tutorials or walkthroughs inside Excel
- Saving online banking data as evidence without leaving Excel

**How to use:**
1. Open the window you want to capture (e.g. online bank portal)
2. Go to Excel → **Insert tab** → click **Screenshot**
3. Select the open window thumbnail
4. The screenshot pastes directly into your Excel cell

> 💡 **Why is this better than the Snipping Tool?** The Snipping Tool sometimes closes the window you are trying to capture. The Excel screenshot function captures it without disrupting the active window.

## 11 · 🔃 Sorting

> **Sort data by one or multiple columns at once**

**When to use:**
- Organize a large data table alphabetically or by date
- Sort by multiple columns (e.g. first by region, then by product)

**Steps:**
1. Select the full data table
2. Go to **Home** → **Sort & Filter** → **Custom Sort**
3. Set Column = `Manufacturer`, Sort On = `Cell Values`, Order = `A to Z`
4. Click **Add Level** to add a second sort (e.g. by Product Name)
5. Click **OK**

**Example Data Table:**

| Date | Year | Territory | Product | Manufacturer | Sales Value |
|---|---|---|---|---|---|
| Jan 1, 2012 | 2012 | East | Toyota-002 | TOYOTA | 3,500,000 |
| Jan 2, 2012 | 2012 | North | Mitsubishi-001 | MITSUBISHI | 2,500,000 |
| Jan 6, 2012 | 2012 | East | Tata Car-001 | TATA | 1,000,000 |

> 💡 **Pro Tip:** Always sort on the full table — do not sort a single column or it will misalign your rows!

## 12 · 🧹 Remove Duplicate Values

> **Find and remove repeated records from your data**

**When to use:**
- You exported data from a system and it has duplicate rows
- You need a list of unique values (e.g. unique city names, unique product types)

**Steps:**
1. Highlight the column with duplicate values
2. Go to **Data tab** → click **Remove Duplicates**
3. Select the column(s) to check for duplicates
4. Click **OK** — Excel removes all duplicates and tells you how many were removed

**Example — Unique City List:**

Before:
```
alabama, california, california, florida, alabama, washington, florida...
```
After removing duplicates:
```
alabama
california
florida
washington
```

> ⚠️ **Warning:** This permanently deletes the duplicate rows. Always work on a copy of your data first!

## 13 · 📐 Text to Columns

> **Split data that is combined in one cell into separate columns**

**When to use:**
- You receive exported data where name and phone number are in the same cell
- Dates and codes are combined together
- You need to split any data that has a consistent separator (comma, colon, space, etc.)

**Steps:**
1. Highlight the column with combined data
2. Go to **Data tab** → click **Text to Columns**
3. Select **Delimited** (for data separated by a character)
4. Choose your separator: Tab, Semicolon, Comma, Space, or **Other** (e.g. colon `:`)
5. Click **Finish**

**Example — HR Name & Phone Split:**

| Before (Combined) | After — Name | After — Phone |
|---|---|---|
| `Mark Henry : 905-213-3008` | Mark Henry | 905-213-3008 |
| `Jeff Henry : 903-424-2343` | Jeff Henry | 903-424-2343 |
| `Alex Jacob : 234-532-3423` | Alex Jacob | 234-532-3423 |

## 14 · 🎨 Conditional Formatting

> **Automatically highlight cells based on their values**

**When to use:**
- Highlight sales below a target in red
- Colour-code performance (green = good, yellow = warning, red = bad)
- Visually identify trends in a large data table

**Steps — Highlight Cells Less Than a Value:**
1. Select the data column you want to format
2. Go to **Home** → **Conditional Formatting** → **Highlight Cell Rules** → **Less Than**
3. Enter your threshold value (e.g. `4,000,000`)
4. Choose a format style (e.g. Light Red Fill with Dark Red Text)
5. Click **OK**

**Popular Rules:**
| Rule Type | Where to Find |
|---|---|
| Greater than / Less than | Highlight Cell Rules |
| Top 10 / Bottom 10 | Top/Bottom Rules |
| Color scale (heat map) | Color Scales |
| Data bars | Data Bars |
| Custom formula-based rule | New Rule → Use formula |

> 💡 **Pro Tip:** Use `New Rule → Use a formula` for advanced rules like: *"Highlight the entire row if column B = 'Overdue'"*
